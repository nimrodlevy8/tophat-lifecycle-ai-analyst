# Data & Insights Deck Template

Source: Monopoly GO! Data & Insights Deck Template (October 2025, Angel Berian)

## Minimum Guidelines

1. Always include creator's name and date on title slide
2. Add direct links to source data (Looker, spreadsheets, Hex) for every visual or table
3. Include the Traffic Light System in the annex
4. Use Monopoly GO! color palette for all visuals

## Color Palette

**Primary (draw attention):** #df2728 (red), #3b3447 (dark purple), #63d200 (green), #dbc585 (gold), #CE9AFF (light purple), #625C45 (olive)

**Secondary (create contrast):** #F8F5E5 (cream), #B3E4BF (light green), #CACACA (gray), #F7F6F3 (off-white)

## Deck Structures by Analysis Type

### Standard Analytics Deck
1. **Executive Summary: The Core Insight** (1 slide)
   - State the single most important takeaway
   - Should stand alone even if rest of deck isn't read
   - "If you remember just one thing from this deck, this should be it."

2. **Deep Dive: The Evidence Behind the Insight** (as many slides as needed)
   - Present the analysis journey with clear visuals and concise takeaways
   - Focus on clarity, not comprehensiveness
   - "This is the 'how' — the data and reasoning that led us to the insight."

3. **Recommendations: What to Do With the Insight** (1 slide)
   - Translate insight into action, prioritized
   - Be clear about implications and possible decisions
   - "Now that we know this, here's what we should do."

4. **Annex: Supporting Details & FAQs** (optional)
   - Additional analysis, edge cases, backup slides
   - Address likely stakeholder questions without cluttering main narrative

---

### Exploratory Analysis Deck
1. **Executive Summary: The Key Discovery** (1 slide)
   - Summarize the most important insight uncovered
   - Frame the "so what" clearly — even if open-ended
   - Mention if further validation is required

2. **Methodology: How We Explored the Question** (1 slide)
   - What was measured, how comparisons were made, assumptions/limitations

3. **Deep Dive: What We Found** (no slide limit)
   - Data patterns, trends, hypotheses uncovered
   - Organize logically around sub-questions
   - Call out signal vs. noise clearly

4. **Recommendations & Next Steps: What We Should Do**
   - Most relevant next actions
   - Be explicit about confidence level and what needs follow-up

---

### Performance Read Deck
1. **Executive Summary: How Did It Perform?** (1 slide)
   - Was it successful? Compare to expectations/benchmarks
   - Topline KPI snapshot: met, exceeded, or fell short

2. **Context & Setup: What Was Run and Why** (1 slide)
   - Purpose, goals, mechanics, target segments, timeline
   - Variations from previous runs

3. **Deep Dive: What Worked, What Didn't** (no slide limit)
   - Break down key metrics vs past events or control groups
   - Segment by player type, region, cohort
   - Highlight surprises, successes, underperformance

4. **Recommendations: What to Double Down On or Fix** (1 slide)
   - What to repeat, scale, tweak, or drop
   - Hypotheses worth testing in future runs

---

### A/B Test Read Deck
1. **Executive Summary: What Did the Test Tell Us?** (1 slide)
   - Main conclusion: variant win, neutral, or inconclusive
   - Short recommendation: roll out, iterate, or stop

2. **Test Setup: What We Tested and Why** (1 slide)
   - Hypothesis, variants, primary metric(s)
   - Allocation, targeting, duration, business context

3. **Results: What Happened in the Data** (no slide limit)
   - Outcomes for primary and secondary KPIs
   - Statistical significance, effect sizes, confidence intervals
   - Trade-offs or unintended effects, segment results

4. **Interpretation: What It Means for the Business** (1 slide)
   - Go beyond stats — practical implications
   - Support or refute hypothesis? Explain unexpected results
   - Was there enough power?

5. **Recommendations & Next Steps: What We Do Now**
   - Course of action with confidence level
   - Risks, caveats, follow-up analyses

---

### New Measurement Framework Deck
1. **Executive Summary: Why We're Updating How We Measure** (1 slide)
   - What's changing and why, anchored in business value

2. **Limitations of the Current Framework** (1 slide)
   - Concrete issues with old approach

3. **Overview of the New Framework** (1-2 slides)
   - Structure: new KPIs, attribution logic, segmentation, methods
   - Define what success looks like

4. **Application Example: Before vs After** (1-2 slides)
   - Walk through real case showing different insights

5. **Rollout Plan: How We'll Transition**
   - Implementation, dashboards, timelines

## Traffic Light System

Communicates **how much trust to place in the insight** — NOT whether the result is good or bad for the business.

### Red Question Mark — Just Started
- **Meaning:** Insight is preliminary and directional. Early signal, not confirmed finding.
- **When:** Start of investigation. Identified signals/symptoms, not validated root causes.
- **Stakeholder expectation:** Active investigation. Regular updates as evidence grows.

### Orange Hammer — Ongoing Investigation
- **Meaning:** Gaining confidence. Emerging evidence pointing to likely cause, but validation still underway.
- **When:** Middle stage. Explored patterns, formed hypotheses, identified potential drivers needing deeper testing.
- **Stakeholder expectation:** Narrowing in on cause. Deep dive outlining what went well/didn't and underlying causes.

### Green Check — Finished Investigation
- **Meaning:** Analysis complete, findings robust. Confident in insight and recommendations.
- **When:** Conclusion. Hypotheses validated, results stress-tested.
- **Stakeholder expectation:** Clear, actionable business recommendation based on sound evidence.

## Visualization Slide Structure

Each visualization slide should contain:
1. **Slide Title** — The slide's biggest callout (the insight, not a description)
2. **Graph Title** — KPIs being shown
3. **The visualization** — Chart/graph with proper colors
4. **Supporting Message** — Commentary explaining the chart AND conclusions
5. **Source Data** — Definitions, filters, and periods used

### Layouts Available
- Full-width chart + supporting message below
- Chart with callout boxes overlaid on data points
- Chart (2/3 width) + supporting message card (1/3 right)
- Side-by-side charts with individual supporting messages

### Best Practices
- Only include events relevant to the analysis on charts
- Use light secondary colors for event overlays (they provide context, not focus)
- Use Area series for multi-day events (minigames), Column for single-day events (holidays, launches)
