# Past Analysis Reference

Distilled summaries of completed analyses. These serve as reference for how we structure analysis, what we've already learned, and what approaches worked.

## How to Add Past Analysis

For each analysis, create a file (e.g., `occasionals_deep_dive.md`) with:

```markdown
# Analysis Title
**Date**: YYYY-MM-DD
**Author**: Name
**Type**: deep-dive / ab-test / research / measurement-framework
**Stakeholder**: Who requested it

## Question
What were we trying to answer?

## Key Findings
- Finding 1 (with key numbers)
- Finding 2
- Finding 3

## Methodology
- What data did we use? (tables, date ranges)
- Key queries or approaches
- Segment cuts applied

## Recommendations Made
- What was recommended?
- What was actually done?

## Outcome
- Did the recommendation get implemented?
- What was the result?
```

## What to Include
Don't copy entire decks — distill each into the key findings, methodology, and what we learned. The goal is to help the analyst:
1. Avoid re-doing work that's already been done
2. Know what approaches worked for similar questions
3. Understand what the team has already learned about a topic
